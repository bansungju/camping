# 캠핑 카테고리 × 평가항목 마스터 설계도

> 목표: 캠핑 장비를 카테고리로 나누고, 각 카테고리마다 **정량 평가항목**을 정의해
> **세그먼트(카테고리×인원/용량) 내 min-max로 별 5개** 환산.
> 텐트에서 검증된 패턴(EAV 메트릭 + 정의통일 + 검증 + 크로스소스)을 전 카테고리로 확장.

별점 공식(공통): `stars = round(1 + 4×p, 0.5)`, p = 세그먼트 내 백분위(방향 보정).

---

## 0. 공통 메트릭 (거의 모든 카테고리)
| key | 항목 | 단위 | 방향 | 출처난이도 | 정의통일 규칙(텐트 교훈) |
|---|---|---|---|---|---|
| **price** | **가격** | 원 | 낮을수록★ | 🟢다나와 | ⭐**필수(MANDATORY)** — 국내정품 기준(직구 channel 분리). 가격 없으면 카탈로그 제외 |
| weight_min | 무게 | g | 낮을수록★ | 🟢다나와 | **최소무게 기준**(패킹무게와 혼입 주의) |
| packed_volume | 패킹부피 | cm³ | 낮을수록★ | 🔴희소 | 원통=πr²h / 박스=곱. 측정법 태그 |

> **제1규칙: 모든 제품에는 가격이 있어야 한다.** 수익모델=가격/링크 클릭이라, 가격 없는 항목은
> 구매로 이어질 수 없어 무의미. → verified 승격 조건에 가격 EXISTS 강제. 없으면 pending+플래그.

🟢=다나와에 흔함 / 🟡=일부 / 🔴=크로스소스 필요

## 1. 분류 체계 (대분류 → 카테고리 → 세그먼트)

```
캠핑용품
├─ 쉘터        : 텐트(백패킹/오토캠핑/기타), 타프, 그늘막
├─ 수면        : 침낭, 매트, 베개, 코트(야전침대)
├─ 취사        : 버너, 코펠/쿡웨어, 아이스박스, 테이블
├─ 가구/거주   : 체어, 테이블, 화로대
├─ 전원/조명   : 랜턴, 파워뱅크/파워스테이션
└─ 운반        : 배낭, 더플/기어백, 웨건
```
세그먼트 키: 텐트/침낭/매트=**인원**, 배낭/쿨러/코펠=**용량**, 나머지=단일.

## 3. 정량 vs 정성 (2개 트랙)
- **정량(measured)**: 위 표의 숫자 메트릭 → 출처에서 수집 → min-max 별점. (현재 파이프라인)
- **정성(scored)**: 편의성·내구성·등판감 등 → **리뷰 텍스트를 LLM이 1~5 스코어링**(confidence 표기). 처음 기획서의 "편의성" 항목이 여기. 별도 트랙으로 추가 예정.

## 4. 카테고리 추가 절차 (텐트에서 확립)
1. `reference.sql` 에 category + metrics(이 문서의 표) INSERT
2. `run_all.py` CATEGORIES 에 블록 추가: queries, core(완비기준), need_capacity
3. (수집) 검색 spec_map 매핑 + 타당범위 + 크로스소스 화이트리스트
4. `python3 pipeline/run_all.py --harvest --only <카테고리>`

## 5. 출처 전략 요약 (난이도별)
- 🟢 다나와: 무게·가격·용량·밝기·출력 등 → 자동
- 🟡 일부: 화력·사용시간·방수 → 다나와+크로스소스
- 🔴 희소(크로스소스/리뷰 필수): 패킹부피·R값·보냉시간·끓임효율 → 메이저만 회수, 나머지 정직하게 빈칸

## 한 줄
**카테고리마다 비교축이 다르다 → config(이 문서)가 곧 제품.** 숫자 수집·별점 환산 엔진은 공용,
카테고리별 "평가항목 정의"만 갈아끼우면 캠핑 전 영역으로 확장된다.
