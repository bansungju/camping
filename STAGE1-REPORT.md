# 1단계 최종 보고서 — 데이터 정합성 & 한계 파악

> 종료 판정: 2026-06-09 · 방법론: 적대적 의심-보완 루프 37라운드 · 결과: 누적 60건 수정 + 1건 근거기각
> 산출물 DB: `camping_tents500.db` · 한계지도: `LIMITS.md`(자동생성) · 파이프라인: `pipeline/`

---

## 1. 목표와 원칙

**목표.** 캠핑용품을 카테고리로 나누고, 각 카테고리의 정량 지표를 세그먼트 내 별점(★)으로 환산하되 — **데이터가 신뢰할 수 있는지, 한계가 어디인지 명확히** 한다.

**불변 원칙(전 과정 관통).**
- **측정값만.** 확인 안 되는 값은 채우지 않는다(추측 금지).
- **브랜드만.** 노네임·부속품·셀러상품은 비교 대상에서 제외(해자).
- **회수 가능 결측은 회수, 미공개 결측은 한계로 명시.** "데이터 없는 건 괜찮다, 틀린 건 안 된다."
- **멱등.** 파이프라인은 몇 번을 돌려도 같은 결과.

## 3. 방법론 — 적대적 의심-보완 루프

자기검증의 사각지대를 없애기 위해 **역할을 분리**했다.

```
[의심 사이클]  새 서브에이전트가 데이터를 적대적으로 공격(의심 제기, 수정 금지)
        ↕     (큐 상태로 자동 교대)
[보완 사이클]  메인이 각 의심을 쿼리로 검증 → 진짜면 수정, 아니면 근거와 함께 기각
```

- 핸드오프 파일 `doubt_queue.md`(OPEN 있으면 보완, 없으면 의심) / 전 이력 `doubt_log.md`
- 수렴 기준: **2회 연속 "새 의심 없음"**. 한 번이라도 발견되면 카운트 리셋 → 다시 2회.
- 37라운드 동안 발견의 무게가 *대량 구조결함 → 단일 이상치 → 표시중복(별점영향 0)* 으로 수렴.

**이 방식이 잡은 것 중 메인이 혼자선 못 잡았을 사례**
- 별점 분모에 rejected(노네임) 2,170종이 섞여 있었음(28R) — "브랜드만" 원칙이 27라운드 동안 깨져 있던 것
- 본인 수정의 부작용 추적: 코트텐트 재분류(8R), floor 진자수렴(15→16R), 지속격리 부활(24R)
- 의도된 설계를 버그로 오해한 공격을 메인이 **근거로 기각**(17R 국내가 우선)

## 4. 수정 60건 — 테마별 요약

**별점 엔진 (가장 중대)**
- min-max 선형 → **세그먼트 내 순위백분위**(9R): outlier 1개가 전체를 왜곡하던 결함 제거, 중앙값=★3
- 모집단을 **verified-only**로(28R) + **canonical(값별) dedup**으로(34~36R): 노네임·변형중복이 분모를 부풀리던 것 근절
- 평행 엔진(catalog_scores, 구 min-max·소비처0) 제거(29R) → 단일 진실원 `ratings`

**세그먼트 설계**
- 텐트=인원별, 랜턴=광원형태(면조명/핸드헬드 분리), 그 외 단일풀
- packed_volume은 참고지표로 강등(별점 제외)

**파서 가드(미래 데이터도 자동 교정)**
- capacity_l(ml/L·음수), capacity_mah(자릿수), floor(이너·테이퍼 평균폭), 가격(채널혼입·이상치), 코펠 무게합산 등

**변형 일관성(같은 모델 색상 SKU)**
- capacity 다수결·NULL 형제채움, weight 최소통일, water_head 다수결, 공백표기 통일

**정직성 장치**
- footprint floor → 외형기준 배지 + 별점제외(이너 측정만 별점)
- 기준오염/모순 → 지속격리(검증 때마다 부활하던 구조버그까지 차단, 24R)
- 미공개 결측 → 한계 명시(LIMITS.md 자동)

**구조·멱등**
- promote 후 recompute 재실행(32R)으로 단일 run_all 멱등 보장
- 외래키 무결성·동점 처리·percentile 범위 전수 검증 통과

> 전체 60건의 라운드별 상세 내역은 `doubt_log.md` 참조.

## 5. 한계 — 정직하게 남긴 것 (수정 불가/안 함)

| 종류 | 내용 | 처리 |
|---|---|---|
| 미공개 결측 | 매트 R값 7%·코펠 용량 30%·버너 화력 44%·침낭 충전량 45%·랜턴 무게 44%·파워뱅크 용량 19% | 한계지도 명시, 해당 축 '데이터부족' 배지 |
| footprint 외형값 | 오토/기타텐트 바닥의 다수가 본체 외형(이너 미상) | '외형기준' 배지 + 별점 제외 (이너 측정 28%만 별점) |
| 기준 불명 | 다나와 무게/내수압은 min·packed 기준 불명 | source_id로 확정/참고 구분 배지 |
| genuine 변형차 | 같은 모델인데 실측 원문이 다른 4건(리빙쉘 등) | 통일하지 않고 각자 제값으로 채점 |

**핵심:** 위 한계는 *결함이 아니라 데이터 가용성의 사실*이다. 사이트는 이를 숨기지 않고 배지·경고로 드러낸다.

## 6. 재현 방법

```bash
python3 pipeline/run_all.py --db camping_tents500.db
# → 정규화 → 검증/격리/별점 → 승격 → 한계지도(LIMITS.md) → 사이트데이터(site/data/*.json)
# 멱등: 몇 번을 돌려도 verified 2324 · ratings 4941 동일
```

**결론:** 1단계 목표("정합성 + 한계 파악")는 달성됐다. 데이터는 신뢰할 수 있고, 신뢰할 수 없는 부분은 명시돼 있다. 2단계(사이트)의 토대로 충분하다.
