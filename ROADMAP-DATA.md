> 작성 2026-06-13. 데이터 수집 4방법(다나와·제조사웹·수동큐레이션·OCR) + 검증체계 구축 직후.
> 목적: 다나와 단일의존 탈피 + 데이터 정확성 검증 체계화. 정직성("측정값만") 유지.
> 관련: VALUE-METRIC-DESIGN.md, 메모리 data-sourcing-methods.

## 실행 순서 / 의존성
```
1주차: (A1 + C1) 즉시 → A2 스캐너
2주차: A3 검증루프 가동(카테고리별 배치) + B1 모델추가
3주차~: B2/B3 제외카테고리 보강·재승격, A4/C2 점진개선
```
- A2(스캐너) → A3(OCR검증)의 입력. B2(핵심지표) → B3(재승격)의 전제.
- A1·C1은 독립 → 병렬 가능. **P0 착수: A1·A2·C1.**
