# 쿠팡 파트너스 자동 링크 생성 런북 (API 발급 전 임시)

> 목적: 구글 시트 `gear-forest-list`의 `제품명`(B열)으로 쿠팡 파트너스 딥링크(`link.coupang.com/a/…`)를 자동 생성해 D열(쿠팡링크)·가격열을 채운다.
> 쿠팡 파트너스 Open API는 누적매출 15만원 후 발급 → 그 전까지의 임시 수단. 관련: 메모리 `coupang-partners-monetization`, `coupang-partners-link-api`.

---

## 1. 실행 방식 (가장 안전한 경로)

- 사용자 크롬의 **로그인된 partners.coupang.com 탭 안에서** 동일출처 fetch/XHR로 내부 API 호출.
  - 도구: `Control_Chrome` MCP `execute_javascript` (CLI 세션에서도 동작. Claude-in-Chrome 확장은 CLI와 페어링 안 됨).
  - 이유: 쿠키·TLS 지문·`cf_clearance`·Akamai(`bm_*`,`_abck`) 전부 실세션 그대로 → **봇차단·토큰만료 회피**. Python 복붙 재현은 TLS 지문 불일치 + `CT_AT` JWT 4시간 만료로 비권장.
- 인증: 동일출처라 쿠키 자동 전송 + 헤더 `x-token` = 쿠키 `AFATK` 값(`document.cookie`에서 읽음). `lang: ko-KR`, `content-type: application/json`.

### 내부 엔드포인트
- **검색** `POST /api/v1/search`
  body: `{"page":{"pageNumber":0,"size":36},"filter":"<키워드>","deliveryTypes":[]}`
  응답: `data.products[]` — 각 항목 `productId,itemId,vendorItemId,title,image,originPrice,salesPrice,travel,type,valid,isSoldOut,brands,ratingCount,badges`.
- **딥링크** `POST /api/v1/banner/iframe/url`
  body: `{"product":{"type":"PRODUCT","itemId":N,"productId":N,"vendorItemId":N,"image":"...","title":"...","originPrice":N,"salesPrice":N,"travel":"false","discountRate":null}}`
  응답: `data.shortUrl`. ⚠️ ID 3개만으론 rCode 400 — title/image/price까지 **전체 payload 필수**.

## 2. 검색 매칭의 함정 (이 런북의 핵심)

**쿠팡 검색은 결과가 없어도 무관한 상품을 퍼지 매칭해서 돌려준다.** 따라서 "검색 1순위 무조건 채택"은 **틀린 링크**를 만든다. 실측 사례:

| 입력 키워드 | 단순 1순위 결과 (❌) |
|---|---|
| 니모이큅먼트 프론트 포치 2P | "해피슈가 슈가글라이더 … 햄스터 장난감" 7,800원 |
| 니모이큅먼트 웨건탑 6P 2020 | "BLACKDOG 마운틴 2.0 캠핑 웨건" (다른 브랜드) |

원인 두 가지:
1. **브랜드 표기 불일치**: 쿠팡엔 `니모이큅먼트`가 아니라 `니모`(NEMO)로 등록 → 풀네임 검색 시 미스.
2. **모델코드/수식어 과다**: `스노우피크 랜드스테이션 L TP-820`은 모델코드 `TP-820`을 빼고 `스노우피크 랜드스테이션 L`로 검색해야 결과가 나옴.

→ **반드시 (a) 브랜드 별칭 정규화 + (b) 키워드 축약 사다리 + (c) 결과 검증**을 적용한다.

## 3. 검색 알고리즘 (반드시 이 순서)

보정 데이터: `pipeline/coupang_search_aliases.json`

1. **오버라이드 우선**: `query_overrides`에 제품명이 있으면 그 검색어를 사용.
2. **브랜드 별칭**: 제품명 앞부분이 `brand_aliases` 키와 매칭되면 치환 (예: `니모이큅먼트`→`니모`). 별칭 적용 후의 첫 토큰 = `브랜드`.
3. **축약 사다리(query ladder)** 생성: 풀네임에서 **뒤쪽 수식 토큰을 하나씩 제거**하며 후보 검색어들을 만든다. 제거 대상 토큰:
   - 모델코드: 영문+숫자 혼합 (`TP-820`, `2P`(용량), `V2`(버전)), 순수 숫자 모델
   - 연도: `20\d\d` (`2020`,`2025`)
   - 용량/인원: `\d+P`, `\d+인용`
   - 단, **브랜드 + 핵심 1토큰** 밑으로는 줄이지 않는다.
4. 사다리의 각 검색어로 `/api/v1/search` 호출. (호출 사이 **랜덤지연 8~25초**.)
5. **결과 검증·선별(critical)** — 여러 결과 중 진짜 그 상품을 고르는 단계. **게이트 → 점수 → 신뢰도** 3중.
   - **(A) 게이트** (하나라도 실패 시 후보 탈락):
     - `brandOK` = 상품 `brands[]` 또는 `title`에 쿼리 브랜드 포함. (햄스터 장난감·타브랜드 차단)
     - `coreOverlap ≥ max(1, ceil(핵심토큰수 × 0.5))` (핵심토큰 = 브랜드·모델코드 제외 쿼리 토큰, title에 substring 등장 수).
     - `valid && !isSoldOut`.
     - **부속품 제외**: title에 `폴대/그라운드시트/전용/보관함/파우치/단품/케이스/리페어` 등이 있고 원래 제품명엔 없으면 탈락(or 강한 감점).
   - **(B) 선택**(게이트 통과분 중에서):
     - 1순위 `coreOverlap` 최대 그룹(= 같은 상품의 중복 리스팅들)을 추림.
     - 그 그룹에서 **최저가(salesPrice) 채택**. 쿠팡엔 동일 모델(예 KECX9TO-04)이 여러 판매자·옵션으로 중복 등록되고 가격차가 큼(마리나: 398k~1033k) → 최저가가 정답.
     - **저가 이상치 가드**: 그룹 median의 40% 미만 가격은 부속품/부분품 의심 → 제외. + 부속품 제외어 적용.
     - 동점이면 `ratingCount` 많은 쪽.
     - ⚠️ **C열(기준가)은 선별에 쓰지 않는다** — 데이터 오류가 잦음(실측: 마리나 C=28000 오타). `차이(G)`는 검수 신호로만.
   - **(C) 신뢰도**: 게이트 통과 후보가 1개뿐이고 모델코드 합의가 없거나, 용량 토큰이 있는데 동일 용량 후보가 없으면 `low_confidence` → 채우되 **사람 검수 대상**.
   - 게이트 통과 0개 → 이 검색어 실패(다음 사다리로).
6. 사다리 한 단계에서 채택되면 종료. 안 되면 다음(더 짧은) 검색어로.
7. **끝까지 게이트 통과 없음 → `x`로 표기(미발견).** ❗절대 미검증 상품으로 링크 만들지 않는다(틀린 링크 < 빈칸/`x`). 미발견 행엔 검색 1순위 raw 제목을 `hint`로 남겨 사람이 override 판단.

> 현재 러너(v2)는 게이트(brandOK+overlap) + overlap 정렬까지 구현. **가격근접도·부속품제외·신뢰도(B/C)는 다음 배치부터 반영** 예정(C열 값을 행별로 함께 로드).

> 새 실패/오매칭을 발견하면 → `coupang_search_aliases.json`의 `brand_aliases` 또는 `query_overrides`에 추가. 코드 수정 불필요, 다음 실행부터 반영.

## 4. 안전장치 (사용자 강조 — 계정 제재 방지, 절대 준수)

- 상품 1건당 호출: 검색 1~N(사다리) + 딥링크 1.
- 모든 호출 사이 **랜덤지연 8~25초** (고정 간격 금지).
- 5건마다 **1~3분 긴 휴식**.
- **일일 상한 기본 10건** (보수적; 필요시 20~30까지).
- 비-`0` rCode / HTTP 429 / 네트워크 오류 → **그날 즉시 중단**(재시도 금지).
- **멱등**: D열이 이미 채워졌거나 `x`인 행은 건너뜀.
- 자동 프로브 난사 금지(auto-mode classifier가 차단).
- **봇처럼 보이지 않기(사용자 강조)**: ① 처리 순서 랜덤화(시트 위→아래 순차 금지) ② 지연 분포를 불규칙하게(균일 간격 금지, 가끔 짧게·가끔 길게) ③ 고정 스케줄/같은 시각 매일 반복 금지 ④ 항상 사용자 실브라우저 세션 안에서 실행(지문 동일).

## 5. 시트 I/O

대상: `gear-forest-list` (id는 env `GEAR_FOREST_SHEET_ID` — 공개 레포 미노출)
열: A 카테고리 · B 제품명(=키워드) · C 가격 · **D 쿠팡링크(=채움/`x`)** · E 쿠팡가격 · F 쿠팡가격(숫자) · G 차이(수식) · H 적용완료

- **읽기**: docs.google.com 탭에서 `gviz/tq?tqx=out:csv&gid=0`(또는 `out:json`) 동일세션 fetch. D 빈 행만 대상.
- **쓰기**: Sheets API(gspread) 미셋업 → **붙여넣기용 TSV 출력**(행번호·D·가격) → 사용자가 시트에 붙여넣기. (※ 가격을 E/F 중 어디에 넣을지: 기존 행은 F(숫자)에 값·`차이`=(F-C)/C, E는 빈칸. 운영 확정 필요.)

## 6. 검증/품질 체크

- 채택 결과는 `matchedTitle`·`price`를 같이 기록 → 사람이 빠르게 검수.

- 생성된 `shortUrl`은 AF6034597 추적 유효 링크.

## 8. 예외 처리 루프 (지속 업데이트 — 핵심)

예외는 계속 쌓인다(빈 행 2천+). "한 번에 완벽"이 아니라 **루프로 점진 개선**한다.

```
배치 실행 → 예외 자동 수집(coupang_exceptions.jsonl) → 유형 분류 → 수정 반영(aliases.json/로직) → 해당 행 재실행 → 해결분 백로그서 제거
```

### 예외 유형 분류표 (새 예외 = 이 중 하나로 분류 후 대응)
| 유형 | 증상 | 대응 | 예시 |
|---|---|---|---|
| **A. 브랜드 표기 불일치** | 풀네임 검색 시 미스/오매칭. 쿠팡은 다른 브랜드명 사용 | `brand_aliases`에 추가 | 니모이큅먼트→니모, 도플갱어아웃도어→DOD |
| **B. 모델코드·수식어 과다** | 뒤에 모델코드 붙으면 결과 0 | 축약 사다리 자동 처리, 안 되면 `query_overrides` | 스노우피크 랜드스테이션 L TP-820 |
| **C. 용량·변형 불일치** | 본체는 맞는데 인원/사이즈 다른 변형 채택 | **용량 인지 선별**(쿼리 `\d+P`/`\d+인용` 있으면 동일 용량 우선, 없으면 low_confidence) | 초고리 3P→2P |
| **D. 쿠팡 미등록(진짜 없음)** | 어떤 검색어로도 정품 없음 | `x` 확정. 필요시 `skip_keywords` | 코베아 로보카 폴리 와우 |
| **E. 띄어쓰기·표기차** | 에코소울 vs 에코 소울 등 | `query_overrides`(정규화된 검색어) | 도플갱어 에코소울 |

### 1배치 운영 절차
1. 러너로 N건 실행 → 결과에서 `ok`는 시트 기록, `not_found`/`low_confidence`는 **`coupang_exceptions.jsonl`에 append**(date, sheetRow, name, triedQ[], hint).
2. 예외를 위 표로 분류. A/B/E면 `aliases.json`에 한 줄 추가, C는 로직(용량) 보완, D는 `x` 확정.
3. 수정된 행만 재실행. 시트에 `x` 이미 박혔으면 그 셀 비우고 재시도.
4. 해결되면 해당 줄을 `coupang_exceptions.jsonl`에서 제거(또는 `resolved` 표기).
5. 새로 발견한 **재사용 가능한 규칙**(예: 특정 브랜드 일괄 romanization)은 이 표/별칭에 반영.

> 목표: 반복할수록 `aliases.json`이 두꺼워지고 예외율이 떨어진다. 사람은 분류·판단만, 적용은 데이터파일이.
