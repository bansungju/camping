## 1. 스토어 정책 / 리뷰 가이드라인

- ⬜ Apple: 4.2 최소기능 / 4.3 스팸(중복·저품질) 방어 근거 확보 (→ B-1)
- ⬜ Apple: 4.8 Sign in with Apple (→ B-2)
- ⬜ Apple: 3.1.1 인앱결제 — 물리상품/제휴 외부결제는 IAP 불요임을 확인, 디지털재화 없음 확인
- ⬜ Apple: 5.1.1(v) 계정 삭제 경로 (→ B-4)
- ⬜ Apple: 1.x 사용자생성 콘텐츠(커뮤니티) — 신고/차단/약관동의 모더레이션 필요. ※ 현재 GNB·커뮤니티 비활성화 상태라면 "기능 없음"으로 일관 처리 (숨겨진 진입점 없는지 확인)
- ⬜ Google: 정책 — 가격/재고 정보 정확성, 오해소지 메타데이터 금지
- ⬜ Google: 타겟 API 레벨 최신 요구치 충족 (제출 시점 기준)
- ⬜ Google: 데이터 보안 섹션 신고 정확성
- ⬜ 양쪽: 18세 미만 타겟 아님 명확화(연령 등급), 광고/제휴 표기

## 2. 패키징 / 기술 빌드 (Capacitor 기준)

- ⬜ Capacitor 프로젝트 구성, `gear-forest.com` 로드 방식 결정 (원격 URL vs 번들 동기화)
- ⬜ iOS: Xcode 빌드, 서명, App Transport Security(HTTPS 전수) 통과
- ⬜ Android: AAB(App Bundle) 빌드, 서명키(Play App Signing) 등록
- ⬜ 외부 링크(쿠팡 등) → 인앱 WebView가 아닌 **시스템 브라우저/SafariViewController**로 오픈
- ⬜ 딥링크/유니버설 링크 (상품 상세 공유 시 앱으로 복귀)
- ⬜ 스플래시 스크린 / 앱 아이콘 / 상태바 네이티브 처리
- ⬜ 오프라인/네트워크 끊김 시 빈 화면 대신 폴백 UI
- ⬜ Service Worker ↔ 네이티브 캐시 충돌 점검 (구 SW 고착 이슈 이력 있음 — `www/sw.js` 리다이렉트 사고 재발 방지)
- ⬜ Google OAuth 리디렉트 URI가 네이티브 셸(커스텀 스킴/유니버설 링크)에서 동작
