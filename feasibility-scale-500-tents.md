# 스케일 테스트: 텐트 500종 수확 (2026-06-07)

**목표**: 텐트 한 카테고리를 500종 규모로 채워, 대량 수집의 현실성·한계 검증.

## 방법 — 2단계 아키텍처로 전환
개별 상품 500회 요청은 느리고 차단 위험. 그래서:
- **1단계 (수확, breadth)**: 다나와 **검색결과(dsearch)** 페이지가 서버렌더 → 한 페이지(40개)에
  pcode·이름·spec_list·가격이 인라인. ~14페이지로 515종 수집. (`harvest_tents.py`)
- **2단계 (보강, depth)**: 가치 높은 후보(경량 상위)만 **상세 fetch**해 깊은 스펙 채움. (`enrich_details.py`)

## 결과
| 구분 | 수치 |
|---|---|
| 수집 텐트 | **515종** (백패킹 344 / 오토캠핑 100 / 기타 71) |
| 분류 | 다나와 태그로 카테고리 자동 분류 |
| **가격 수집** | **515 / 515 (100%)** ← 검색결과는 최저가를 다 보여줌 |
| 1단계 스펙값 | 643개 (주로 무게) |
| 2단계 보강 후 | 684개 (경량 25종에 내수압·바닥면적 +41) |
| 품질 플래그 | 965건 → 보강으로 일부 해소 |

## 핵심 발견

### 1. breadth와 depth는 출처가 다르다 ⭐ (가장 중요)
- **검색결과**: 무게 + 가격 + 분류만. 넓게(500+) 싸게. **내수압·설치크기 없음.**
- **상품 상세**: 내수압·설치크기·수납크기까지. 깊지만 1건씩.
→ 정답은 **2단계 파이프라인**: 전수 수확(얕게) → 가치 후보만 보강(깊게).
   500개를 다 상세조회할 필요 없음. 무게/가격만으로도 1차 비교·필터 가능.

### 2. 검색결과가 가격 출처로 더 우수 (의외의 반전)
이전엔 "가격이 최약점(상세페이지 절반 없음)"이었는데,
**검색결과 최저가는 515종 100% 수집됨.** → 가격은 검색에서, 스펙 깊이는 상세에서.

### 3. 대량 수집 시 새 데이터 품질 이슈 등장
- **변형 다수**: MIER 색상별 4건, 씨투써밋 그레이/그린 등 동일모델 색상 변형이 별도 등록
  → 모델 정규화(색상 통합) 필요.
- **이상치**: 비비/해먹형이 바닥면적 0.02~0.03㎡로 파싱 (설치크기 포맷 특이/소형)
  → **메트릭별 타당범위 검증**(예: 텐트 바닥 0.5~10㎡ 벗어나면 flag) 추가 필요.
- 자동 분류 정확도: "백패킹용" 태그 기반은 잘 맞으나, 그늘막/팝업 혼입 일부.

## 결론
**텐트 500종 규모 수집은 현실적이다.** 단,
- 무게·가격·분류는 **검색 수확으로 즉시 500+** 확보 가능
- 내수압·바닥면적 등 깊은 스펙은 **2단계 보강**으로 필요한 만큼만
- 다음 QA 과제: **모델 정규화(변형 통합)** + **메트릭 타당범위 검증**

## QA 강화 (후속 작업, 완료)
- **모델 정규화** (`normalize_models.py`): 색상/옵션 변형 통합 → **515종 → 고유 480종**.
  canonical_models 테이블(대표제품/변형수/가격범위) 생성.
- **전체 2단계 보강** (`enrich_details.py --all`): 고유 480종 대표 상세조회로
  내수압·바닥면적 보강 → 스펙값 684 → **1,411개**.
- **타당범위 검증** (`validate_ranges.py`): 카테고리별 물리범위로 이상치 70개 격리(valid=0).
  - 파싱버그 발견: '수납크기'를 '설치크기'로 오인(45x13x13→0.06㎡) → find_spec에 exclude=['수납'] 수정.
  - 범위는 카테고리별이어야 함: 40kg 에어텐트는 오토캠핑에선 정상(백패킹이면 이상치).
  - valid=0 값은 별점 계산서 자동 제외 → 카탈로그 클린 유지.

## 최종 상태 (camping_tents500.db)
- 등록 515 / 고유 480 / 스펙값 1,411 / 이상치격리 70 / 가격 515(100%)
- 클린 카탈로그 동작: 백패킹 2인 텐트 무게+내수압+바닥 비교 (니모 호넷2P 935g 등)
