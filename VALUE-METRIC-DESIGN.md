# 가성비(value) 별점 — 전 카테고리 설계 (확정본)

> 상태: **설계 확정 / 구현 대기**. 백패킹 가방만 선반영(원/L) 되어 있음.
> 목적: "가성비" 별점을 전 카테고리로 확장. 정직성("측정값만·추측 없음") 유지.

## 1. 핵심 공식 (2번 방식 — 다지표 스코어링 ÷ 가격)

```
qᵢ = 지표별 '순위 기반 연속 정규화' [0~1]   (best=1.0, worst=0.0, 등수 균등배분)
Q  = mean(qᵢ)                              (동일가중 평균; 파생 value 지표는 제외)
V  = Q / price                             (클수록 가성비 좋음)
가성비 별점 = V를 카테고리 내 순위 환산 → 5.0(best)~1.0, 0.5 단위
```

- **순위 기반 연속 정규화**: 별점(0.5칸 뭉텅이) 대신 매끄러운 0~1 값으로 평균 → 정밀도 보존, 이상치에 강함. 0.5칸 변환은 **마지막 한 번만**.
- **동일가중**: 자의적 가중치 없음 = 중립 기본값(정직성).
- 단일지표 카테고리는 자동으로 Q=그 지표 → "지표÷가격" 비율로 환원.

## 2. 적용 범위

### 제외 — 가성비 미제공 (5개)
| 카테고리 | 치명적 한계지표(결측 多) | 제외 사유 |
|---|---|---|
| 매트 | R값 (242/260 결측) | 단열=매트 핵심가치. 빼면 가성비 거짓 |
| 보조배터리 | 용량 (55/70 결측) | 용량=핵심가치 |
| 침낭 | 충전량 (126/244 결측) | 보온=핵심가치 |
| 버너 | 최대화력 (126/227 결측) | 화력=핵심가치 |
| 코펠 | 용량 (49/86 결측) | 용량=핵심가치 |

→ 한계지표를 빼면 커버리지 100% 되지만, **그 지표가 제품가치의 본질**이라 제외 시 가성비가 호도. 그래서 **카테고리 통째로 미제공**.

| 카테고리 | 가성비 사용지표 | 표시(F3) | 커버리지 |
|---|---|---|---|
| 가방 | 용량 | **원/L** (현행 유지) | 100% |
| 화로대 | 무게 | **원/g** | 100% |
| 의자·코트·테이블 | 무게·내하중 | 지수+별점 | 100% |

| 왜건 | 무게·적재하중 | 지수+별점 | 93% |

## 3. 부여 조건 (정직성)
- **그 카테고리의 모든 (비파생) 별점지표 값을 가진 모델에만** 가성비 부여.
- 일부 지표 결측 모델 → 가성비 **미부여**(데이터부족). 부분 평균 안 함.
- 결과: 100% 카테고리는 전 모델, 78~93% 카테고리는 풀데이터 모델만.

## 4. 표시 (F3 하이브리드)
| 유형 | 값 표시 | 예 |
|---|---|---|
| 단일지표 | 구체 비율 + 별점 | "323원/L ★★★★★" / "원/g" |
| 다지표 | 가성비 지수(0~100) + 별점 | "가성비 지수 72 ★★★★☆" |

- 상세페이지 안내문(`.spec-note`)을 **카테고리 지표 기반 동적 생성**:
  - 다지표: "가성비 = N개 지표(무게·내수압·바닥면적) 평균 품질 대비 가격. 순위 환산(절대점수 아님)."
  - 단일지표: "가성비 = 용량 1L당 가격(원/L). 낮을수록 좋음."

## 5. 구현 위치 (착수 시)
- **`pipeline/export_site.py`** 에 가성비 후처리 패스 추가 — DB→JSON 생성 직후 각 카테고리 models에 `value_*` 메트릭/specs 주입. (12개 카테고리 일괄, 5개 스킵)
- **`scripts/build-item-pages.js`** spec-note를 카테고리 지표 기반 동적화.
- **가방**: 현행 `value_per_l`(원/L) 유지 — 통합하지 않음.
- 메트릭 키: 단일=구체(`value_per_l`/`value_per_g`), 다지표=`value_score`(지수).

## 6. 미결/주의
- 78~93% 카테고리: 결측이 **비(非)한계지표**(내수압 등)라 일부 모델은 영구 미부여 — 정상.
- 가성비 지수(0~100) 정규화 방식(순위 vs min-max)은 표시용 — 순위 기반 권장.
