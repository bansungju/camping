## P1 — 사용자 가치가 가장 큰 것

### P1-A. 상품 이미지 수집 (KREAM 완성도)
- **무엇**: 다나와 상품페이지의 `og:image`/대표 썸네일 URL을 수집 → DB(products에 image_url 컬럼) → JSON export → 카드·표에 이미지 렌더.
- **왜**: 현재 이모지 아이콘으로 모양만 잡음. 실제 상품 사진이 깔리면 인지·신뢰·전환이 크게 오름(사용자가 KREAM 참조한 핵심).
- **작업 단계**:
  1. `pipeline/`에 이미지 URL 수집기(다나와 상세 fetch에서 og:image 파싱) 추가
  2. variant 혼동 방지 — canonical 대표 1장 기준, 색상변형은 대표 이미지 공유
  3. export_site에 image 필드 추가, app.js 카드/표/추천에 `<img>` (lazy-load)
  4. 호스팅 결정: **핫링크(다나와)** = 용량0·차단위험 vs **자체저장(GitHub)** = 안정·용량↑. 1차는 핫링크로 PoC, 차단되면 자체저장 전환
- **정직성**: 이미지는 사실이라 OK. 단 "그 모델의 이미지가 맞는가" 검증(잘못된 이미지=오정보). 측정 스펙 정직성과 별개.
- **리스크**: 다나와 핫링크 차단/저작권, 이미지 없는 제품(공백 처리=아이콘 fallback), Pages 용량.

### P1-B. 3단계 수익화 — 쿠팡 파트너스
- **무엇**: 각 제품에 쿠팡 어필리에이트 링크(딥링크 또는 검색링크) + 클릭 추적.
- **왜**: 프로젝트 북극성(어필리에이트 수수료). 1·2단계가 견고해진 지금이 적기.
- **작업 단계**:
  1. 쿠팡 파트너스 가입 → API/딥링크 키
  2. 제품→쿠팡 매칭: 정확 매칭 어려우니 **브랜드+모델명 검색 딥링크**가 현실적(클릭 시 쿠팡 검색결과)
  3. 제품 상세/카드에 "쿠팡에서 보기" 버튼, `rel="sponsored"` + 어필리에이트 고지
  4. 클릭 이벤트 측정(GA 등)
- **정직성**: **어필리에이트임을 명시**(고지 의무). 가격 비교 정직성을 해치지 않게 — 추천 순위는 스펙 기준 유지, 링크만 수익.
- **리스크**: 쿠팡 정책·매칭 정확도. 검색링크는 정확도 낮으나 안전.

## P2 — 기능 확장

### P2-A. 페르소나 확장 — 차박 · 부시크래프트
> 기존 추천 엔진(target/value/방향 정렬) 그대로 재사용 → 작업량 작음. 측정 불가 키워드는 노트로 정직 명시.
- **차박/스텔스**: 매트(두께·R값↑=평탄화·보온) + 경량 소품(랜턴·파워뱅크) + 미니 코펠. "도킹"은 측정값 없음 → 노트 명시.
- **부시크래프트**: 타프(텐트 대신) + 화로대 + 튼튼한 코펠(내구 proxy) + 최소 구성. "자연물·날것"은 철학이라 측정 불가 → 노트.
- **정직성**: 두 페르소나는 76~80R 검증 패턴(별점축 일치·동점 가격정렬·측정불가 노트) 그대로 적용.

### P2-B. 데이터 신선도 (가격 stale 관리)
- **무엇**: 가격은 시간이 지나면 변함. 주기적 재수집 + "마지막 갱신일" 표기 + 오래된 항목 경고.
- **작업**: 다나와 가격 재수집 스크립트(주1회 등) + products에 price_updated_at + UI에 "N일 전 가격" 배지(오래되면 흐리게).
- **왜**: 가격 비교가 핵심인데 stale하면 신뢰도↓. 이미 "기준일" 있으나 항목별 신선도는 없음.

## P3 — 품질·정직성 다듬기

### P3-A. 파워뱅크 mAh → Wh 지표 재설계
- **문제**: 최신 파워스테이션(LFP)은 Wh로 표기, mAh 미공개 → 구조적 데이터부족. mAh는 셀 전압 가정이 섞여 비교도 부정확.
- **해결**: Wh를 1차 지표로(또는 병기). 수집 시 Wh 우선, mAh는 보조. → 정직성·비교정확도 동시 개선.

### P3-B. 미세 UX 개선 (묶음, 소작업)
- 추천 카드 가격을 **범위(min~max)** 옵션으로(현재 최저가 단일 — 정직하나 카테고리표와 형식 다름)
- 백패커에 **경량 파워뱅크** 픽(현재 powerbank 경량 표본 빈약 → 표본 보강 후)
- 가족에 **의자** 픽 추가(테이블은 이미 추가됨)
- 정렬 가능 카테고리/지표 라벨 일관성 점검

### P3-C. 데이터 회수 상시 (적대 루프 유지)
- 침낭 충전량·버너 화력은 *진짜* 구조적 미공개로 확정됐으나, **신규 프리미엄 제품** 추가 시 회수 여지.
- 적대 루프(서브=감시/메인=보완) 상시 가동으로 신규 데이터·신규 코드 회귀 감시.

## 보류 — 데이터 선행이 필요한 것

### 감성 페르소나
- 디자인·분위기는 측정 데이터 0 → "추측 금지" 위반.
- **대안**: 브랜드 취향 기반(헬리녹스·스노우피크 등 감성 브랜드군)을 **"취향 기반(측정 아님)" 라벨**로 분리 제공. 측정 추천과 명확히 구분하면 가능.

### 펫 페르소나
- 안전펜스·펫용품 카테고리가 **DB에 아예 없음**.
- **선행 작업**: 펜스/펫매트/펫용품 카테고리 수집 → 정량 지표 정의 → 그 후 페르소나.
